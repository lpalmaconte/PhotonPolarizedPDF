Parte 2. Comparaci√≥n de modelos con el test F.

Vimos arriba que en algunos casos, los t√©rminos de interacci√≥n mejoran el desempe√±o del modelo, mientras que en otros casos, la mejora es poco importante. En general, cuantos m√°s par√°metros tenga el modelo mejor ser√° el ajuste de los datos. Entonces, ¬øpor qu√© no incluir siempre t√©rminos de interacci√≥n y listo? La verdad es que adem√°s de ajustar bien los datos, nos interesa tener el modelo m√°s simple posible, para poder interpretarlo y comunicar los resultados de manera m√°s directa.

Entonces, necesitamos alguna forma de saber cu√°ndo agregar un par√°metro m√°s a un modelo ‚Äúno vale la pena‚Äù. En otras palabras, cu√°ndo la mejora del ajuste no es significativa para la cantidad de par√°metros que se agregaron. Ya mencionamos el R2 ajustado, que tiene en cuenta el n√∫mero de par√°metros. Ahora vamos a introducir una herramienta mucho m√°s rigurosa para comparar modelos, el test F.

Para poder hacer este test, calculamos para cada modelo que queremos comparar la suma del cuadrados de los residuos, \(RSS\), para cada modelo. Es decir, tenemos que sumar los residuos al cuadrado para cada modelo:

\[ RSS = \sum_{i=1}^N \left(y_i - f(x_i)\right)^2\;\;,\] donde \(N\) es el n√∫mero de puntos del dataset, \(y_i\) es el valor de la variable target y \(f(x_i)\) es la predicci√≥n del modelo para ese punto.

Los valores de \(RSS\) para cada modelo se commparan teniendo en cuenta el n√∫mero de par√°metros (lo llamamos \(p\)) para cada modelo. Construimos el estad√≠stico de test (¬°jerga!): \[ F = \frac{\left(\frac{RSS_1 - RSS_2}{p_2 - p_1}\right)}{\frac{RSS_2}{N - p_2}}\;\;.\]

En esta expresi√≥n, \(N\) es el n√∫mero de puntos. Podemos ver que no es lo mismo qu√© modelo ponemos como modelo 1 y cu√°l como modelo 2. Siempre la idea es usar al modelo 1 como el m√°s siemple (el que tendr√° \(RSS\) mayor).

Con este estad√≠stico, que se muestra que tiene una distribuci√≥n conocida, se puede calcular un p-valor, que indica la probabilidad de haber obtenido un valor de \(F\) igual o m√°s extremo que el observado bajo la hip√≥tesis de que ambos modelos son igualmente buenos para describir los datos. Es decir, si esta probabilidad es muy baja, podemos decir que agregar los par√°metros adicionales vali√≥ la pena y que el nuevo modelo captura mejor los datos.

Nota: esta es la visi√≥n cl√°sica de la estad√≠stica, que tiene much√≠simos problemas. Cuando nos volvamos bayesianos, dentro de algunas materias, vamos a presentar estos problemas. Por ahora, usemos esta √∫til herramienta sin preocuparnos por estos temas.

Parte 3. Variables confundidoras

Cuando queremos tomar decisiones basadas en an√°lisis como los que venimos haciendo, tenemos que estar seguros de no estar confundiendo efectos. Por ejemplo, es bien conocido que ‚Äúla correlaci√≥n no implica causalidad‚Äù. Esto quiere decir que si vemos una relaci√≥n entre dos variables, esto no implica necesariamente que una variable sea la causa la otra. Por ejemplo, puede existir una tercera variable (llamada confundidora) que est√© causando ambas variables originales, por lo que vemos una relaci√≥n entre ellas.

Por lo tanto, si queremos ver la intensidad del efecto de una variable sobre la otra, tenemos que tener en cuenta, en la medida de lo posible, potenciales variables confundidoras. Esto se dice controlar por tal variable confundidora.
